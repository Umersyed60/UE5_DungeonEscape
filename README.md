# Dungeon Escape (Unreal Engine)

## Overview
This project is a **first-person puzzle game** built in Unreal Engine where players must interact with objects to escape a dungeon.  
It combines **gameplay design** with **C++ programming and Blueprints**, showcasing my exploration of Unreal’s core systems.

## 🎮 Gameplay
- Navigate a dungeon in first-person view.
- Solve puzzles by picking up and using objects to unlock your exit.
- Designed to demonstrate **interaction mechanics, collisions, and input handling**.

## 🔹 Key Learning & Features
- **GameMode, inheritance & composition** for structured gameplay logic.
- **Actor components, tags & collisions** to build modular systems.
- **Pointers & `CreateDefaultSubObject`** for engine-level programming.
- **Delegates & `FindComponentByClass`** for event-driven interactions.
- **Input system** for player controls and interaction.
- **Line & sphere casting (`SweepSingleByChannel`)** for detecting objects.
- **`FMath::VInterpConstantTo`** for smooth object movement and transitions.

## 🛠️ Tech Stack
- Unreal Engine (C++ + Blueprints)
- Core gameplay framework
- Collision & physics systems
- Lighting and materials setup

## 🚀 Why This Project Matters
This project strengthened my understanding of Unreal’s **gameplay framework** and **low-level C++ integration**, while reinforcing my adaptability across both **Unity (8+ years)** and **Unreal (1 year)**.  
It’s part of my growing portfolio of **game prototypes** designed to demonstrate technical depth and creativity.

## ▶️ How to Run
1. Clone the repository.
2. Open the project in Unreal Engine (5.x recommended).
3. Build and run in editor or package for Windows.
4. Play in first-person mode and solve puzzles to escape the dungeon!
